# 業種別ペルソナ ―― 簿外債務発見エンジン（4業種）

> 既存デモ（運送業・東和ロジスティクス）と**同一のデータ構造**で4業種を追加するためのデータ定義。
> 各業種は `company` / `priors` / `sources` / `flags` を持つ。`flags` の各要素は既存デモの `FLAGS` と同じスキーマ：
> `{id,title,type,conf,confLv,amt:[lo,hi]|null,struct?:{label,value},glance,prior,nodes:[{s,v},{s,v}],gap,flagText,inference,deal}`
>
> `type` は `"矛盾"|"時系列"|"言語"`、`confLv` は塗りつぶすドット数（高=3／中〜高=3／中=2／低=1）。
> 金額は `万円` 単位。各業種の「★目玉」は、その業種に特有でインパクトの大きい発見。

## 共通メモ（実装時）

- 既存デモの運送業（東和ロジスティクス）は `key:"logistics"` として5業種の先頭に置く。
- 各業種の `flags` は7件で統一。`isStar:true` の項目はその業種の目玉（UIで強調してもよいが、まずは同一表示でOK）。
- `struct` を持つ項目（運送#5 CoC、建設#5 JV）は金額レンジではなく「売上影響／要精査」を表示する既存ロジックをそのまま使う。
- `confLv` は塗りつぶしドット数：高=3／中〜高=3／中=2／低=1。
- 金額表示・合計レンジの計算ロジックは既存の `yen()` / `rangeYen()` / `updateSummary()` をそのまま流用（`struct` 項目は合計に含めない）。
