# AXEON DD INTELLIGENCE シナリオパラメータ設計書
サンプル企業データ(20_ma_demo_companies.json)と組み合わせて、「パラメータを変えると提案が変わる」演出を実現するための設計。

---

## 1. 設計の核心:AIに数字を作らせない

パラメータ変更で出力が変わる演出は、AIに自由生成させると数値が毎回ブレて破綻します。原則はこれ:

> **AIの役割は「計算」ではなく「選択と説明」。数値はすべて企業JSONのレンジ内、レバーはJSONのレバー在庫からの組み合わせのみ。**

- EXIT金額の計算はフロント側(JS)で決定的に行う
- AIはその計算結果を受け取り、「なぜこのレバー構成か」「簿外債務をどう処置するか」を語る
- これにより何度デモしても数字が一貫し、パラメータ変更時の差分が明確に見える

## 2. 調整パラメータ(3つ)

| パラメータ | UI | 効果 |
|---|---|---|
| `exit_target` | 金額スライダー(例: 10〜60億円) | 必要EBITDAを逆算し、達成に必要なレバー数が増減。届かない場合は「ギャップ警告」を表示 |
| `horizon` | 期間セレクタ(2年 / 3年 / 5年) | `months <= horizon`のレバーのみ選択可能に。短期ほど即効レバー(restructure系)に偏る |
| `strategy_axis` | 3択タブ: システム導入による効率化 / 不採算事業・部署の整理 / 基本戦略の見直し | レバー在庫の主軸が切り替わり、**簿外債務の処置方針(offbalance_treatment)も連動して変わる** |

## 3. EXIT計算モデル(フロント実装)

```
調整後EBITDA_現状 = ebitda + owner_addback
EBITDA_計画 = 調整後EBITDA_現状 + Σ(選択レバーのebitda_impact中央値 × 達成率)
  達成率 = min(1, horizon ÷ レバーmonths)  ※期間内に完了しないレバーは按分

想定株式価値 = EBITDA_計画 × multiple − net_debt − 簿外債務調整額 + onetime_gain

簿外債務調整額 = Σ(dd_findings.quantitative.estimate) を処置方針で係数調整
  - 是正済み(strategy_axisの処置が該当項目をカバー): 中央値 × 0.5
  - 未処置(価格調整・エスクロー対応): 上限値 × 1.0
multiple = exit.ebitda_multiple の中央値(戦略見直し軸で完了時は上限側へ+0.25補正)
```

- `exit_target`との差額を**EXITブリッジ図**(現状EBITDA→レバー積み上げ→債務控除→想定価値→目標とのギャップ)で常時表示
- ギャップがプラスなら緑、届かない場合は赤+「不足分: レバー追加 or 期間延長 or 目標修正」の3択サジェスト

## 4. strategy_axis 3軸の挙動差(演出の見せ場)

| 観点 | システム導入効率化 | 不採算整理 | 基本戦略の見直し |
|---|---|---|---|
| 使うレバー | valueup_levers.system | valueup_levers.restructure | valueup_levers.strategy |
| 立ち上がり | 中速(6〜15ヶ月) | 最速(3〜12ヶ月)+一時費用 | 遅い(12〜36ヶ月)が上限が最大 |
| 簿外債務への態度 | **恒久是正型**(仕組みで再発を止める) | **切り離し型**(撤退・確定・エスクロー) | **交渉・転換型**(契約再交渉で裾を切る) |
| ロードマップの語り口 | 導入→定着→効果測定 | 意思決定→実行→固定費確定 | 仮説→検証→スケール |
| リスク注記 | 定着しないリスク・CAPEX | 一時費用・人員影響・レピュテーション | 時間リスク・実行力依存 |

**簿外債務の処置文はJSONの`offbalance_treatment[axis]`をそのまま核として使う**。軸を切り替えると同じ「未払残業代45〜80」への対応が「システムで恒久是正」→「拠点集約で構造ごと解消」→「契約再交渉と一体で処理」と変わる——ここがこのデモ最大の演出ポイント。

## 5. AI呼び出し設計

### 入力(フロント→API)
```json
{
  "company": { ...企業JSONの該当1社... },
  "params": { "exit_target": 4000, "horizon_months": 36, "strategy_axis": "system" },
  "computed": {
    "selected_levers": [...フロントで選択済み...],
    "ebitda_plan": 265, "equity_value": [1850, 2400],
    "gap_to_target": -350, "offbalance_adjusted": 95
  }
}
```

### システムプロンプト(要旨)
```
あなたはM&Aアドバイザリーファームのバリューアップ責任者です。
与えられた企業データ・パラメータ・計算結果に基づき、指定JSONで提案を出力してください。

厳守事項:
- 金額・期間は company / computed に含まれる値のみ使用。新しい数値を作らない
- レバーは selected_levers のみ言及。在庫にないレバーを提案しない
- 簿外債務の処置方針は offbalance_treatment[strategy_axis] を核に、dd_findingsの各項目へ具体化する
- gap_to_target が負の場合は達成不能を明言し、①期間延長 ②軸の複合 ③目標修正 の順で提示
- 出力はJSONのみ
```

### 出力スキーマ(表示カードに対応)
```json
{
  "diagnosis": "現状診断 2〜3文",
  "plan_narrative": "選択軸でのバリューアップ方針 3〜4文",
  "lever_details": [ { "lever": "", "rationale": "", "kpi": "" } ],
  "offbalance_plan": [ { "item": "", "treatment": "", "timing": "" } ],
  "exit_story": "想定買い手と訴求ポイント 2〜3文",
  "roadmap": { "phase1": "", "phase2": "", "phase3": "" },
  "gap_advice": "目標未達時のみ。それ以外はnull",
  "risks": ["2〜3件"]
}
```

## 6. UI演出の要点

1. **差分ハイライト**: パラメータ変更で再生成した際、前回と変わったカード(数値・レバー・処置方針)を2秒間ハイライト。「変えたら変わった」を視覚で証明する
2. **EXITブリッジ図**: 滝グラフを常時上部に固定。スライダー操作でリアルタイムに棒が伸縮(これはAI不要、JS計算のみ)→ AI再生成は「提案を更新」ボタンで明示的に実行し、API消費を制御
3. **軸切替は3タブ**: タブを切り替えた瞬間に簿外債務カードの処置文が変わるのが最初の驚き。ここはプリセット表示(JSON直読み)で即時反映し、詳細説明のみAI生成にすると速い&安い
4. **クライアント入力モード（実装済み `/ai`）**: 規模フォーム（売上・利益・従業員・業種・依存度）→ 業種テンプレ継承＋売上比スケール → **prior／データルーム／追加資料 → 解析実行 → 決定的フラグ報告** → その後 EXIT 試算。サンプル経路ではナレッジ UI を出さない
5. **AI 役割分割（実装済み `/ai`）**: findings／金額はフロント決定的。AI は (A) フラグ文言の補強 (B) 解析後の自由質問 (C) バリューアップ語りのみ。いずれも新規推計額禁止
6. **免責表示**: 「本試算はサンプル診断であり、実案件では追加調査が必要です」を全出力に固定(既存画面の文言を踏襲)

## 7. デモ台本(推奨90秒)

1. 運送タブ・東和ロジで開始 → EXIT目標30億・3年・システム軸で「提案を更新」
2. **軸を「不採算整理」へ切替** → 同じ未払残業への処置が変わるのを指差し確認
3. EXIT目標を45億へ → ギャップ警告と「期間延長 or 軸複合」のAI提案
4. 期間を5年へ → 戦略軸レバーが解禁されギャップ解消
5. 締め: 「御社の対象企業データを入れれば、この分析がそのまま実案件の初期検討になります」
