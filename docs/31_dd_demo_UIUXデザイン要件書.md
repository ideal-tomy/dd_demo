# DDデモ UIUX・デザイン要件書
対象: AXEON DD INTELLIGENCE(体験設計書 30_dd_demo_体験設計書.md の実現)
本書はCursorに渡す実装要件。体験設計書が「何を感じさせるか」、本書が「どう見せ・どう動かすか」。

---

## 0. デザインコンセプト

- **主題**: プリンシパル投資家の意思決定室。派手なSaaSではなく「投資委員会に出せる画面」
- **トーン**: 静かな高級感。余白と数字の精度で信頼を作り、色数を絞る。装飾ゼロ、しかしテンプレ感もゼロ
- **シグネチャー(この画面が記憶される一点)**: **「返した時間」カウンター**。画面右上に常駐し、AIが作業を代行するたびに数字が静かにカウントアップする。デモの思想(人間を思考に戻す)をUIそのもので語る唯一の要素として、これだけに演出コストを集中する
- **誠実さの原則(体験設計書 §0-1 と同一)**: 演出してよいのは**見た目と速度**。演出してはいけないのは**能力**。聞かれた瞬間に「どこが本物でどこが演出か」を即答できるUI文言・注記を置く
- **禁止**: 無言のスピナー/3行を超える連続本文/意味のないアニメーション/グラデーション装飾/**能力の偽装**(決定的結果を「いま解析しました」と見せること)

### 0-1. 段階導入(実装の切り方)

全面リデザイン(§1〜2のトークン・二分割ワークベンチ)は**受注後でよい**。商談投入の先行セットは次のみ:

1. 「返した時間」カウンター(根拠付き内訳)
2. 「あなたへの問い」カード(`questions`在庫)
3. シナリオ層の明示注記
4. (商談タイプに応じ) P4戻り状態保持 / または ライブ層実解析1本

既存レイアウトの上に載せてもよい。受け入れチェックリストは「先行」と「全面」を分ける(§6)。

## 1. デザイントークン

### 1-1. カラー(6色+セマンティック)
| トークン | 値 | 用途 |
|---|---|---|
| `--ink` | #101826 | 見出し・主要数値・ヘッダー背景(AXEONネイビー) |
| `--paper` | #F7F8FA | ページ背景 |
| `--surface` | #FFFFFF | カード面 |
| `--line` | #E3E7ED | 罫線(0.5〜1px) |
| `--muted` | #6B7686 | 補助テキスト・ラベル・シナリオ注記 |
| `--brass` | #B8975A | シグネチャー専用(タイムカウンター・フェーズ完了マーク)。**他用途での使用禁止**(希少性が価値) |
| semantic | 達成 #1D8A66 / 注意 #D98E1F / 未達・債務 #C6453F / 情報 #2F6FB2 | 意味がある時のみ。装飾での使用禁止 |

ダークは対応しない(v1)。代わりに`--ink`背景の「委員会モード」ヘッダーで引き締める。

### 1-2. タイポグラフィ
- 本文・UI: **Noto Sans JP** 400/500(600以上は使わない)
- 数値・金額: **font-feature-settings: "tnum"(等幅数字)必須**。金額は欧文数字+単位を50%サイズの和文で(例: `7.2`大+`億円`小)。桁が揃わない金額表示はこのプロダクトでは品質バグとして扱う
- スケール: 12(注釈)/13(補助)/14(本文)/16(カード見出し)/20(セクション)/**28〜36(ヒーロー数値)**。本文とヒーロー数値のコントラストを最低2倍確保
- 行長: 本文は全角38字まで。超える場合は折りたたみ

### 1-3. 形状・余白・モーション
- 角丸: カード12px/コントロール8px/バッジ99px。単辺ボーダー(左線アクセント)は角丸0
- 余白: 4pxグリッド。カード内padding 16-20px、セクション間32px(SP)/48px(PC)
- モーション: 150ms(ホバー)/250ms(展開・タブ)/400ms(数値カウントアップ)。easing: cubic-bezier(0.2,0,0,1)
- **差分フラッシュ仕様**: 変更されたカード背景を#FBF3E2(brass 8%)→透明へ1.2秒フェード+値の右に▲▼バッジを3秒表示後フェード。`prefers-reduced-motion`時はフラッシュ省略しバッジのみ
- ライブ実況のストリームは**見せ方の演出**(可)。能力の証明は結果と根拠行に置く

## 2. レイアウトアーキテクチャ

### 2-1. PC(≥1024px):二分割ワークベンチ
```
┌────────────┬──────────────────────────────┐
│ 左レール 320px │ 右: 結果ストリーム(max 760px)      │
│ ・対象企業カード │ ・フェーズステッパー(上部固定)       │
│ ・シナリオ注記   │ ・サマリーカード(価値/ギャップ)      │
│ ・シナリオ      │ ・各セクションカード                │
│  パラメータ     │                                  │
│ (常時表示・     │ 右上固定: 返した時間カウンター        │
│  即時反映)     │                                  │
└────────────┴──────────────────────────────┘
```
- 左レールはスクロール固定(sticky)。**パラメータと結果が同一視界にある**ことがPC体験の核 — 触ると右が変わるのを目で追える
- 根拠表示はスプリットビュー: 発見事項クリックで右側に原文パネルがスライドイン(該当行ハイライト)
- **シナリオ層注記**(サンプル企業選択時): 左レールまたは結果先頭に12px/`--muted`で  
  `サンプル企業は事前診断済みのシナリオです`

### 2-2. スマホ(<768px):結果ファースト+ボトムシート
- 上部stickyサマリーバー(高さ56px): `株式価値 7.2億 | ギャップ −23億`+ミニカウンター。スクロールしても現在地を見失わない
- パラメータは**ボトムシート**。右下FAB「条件を変える」→シート内で操作→「反映」でシートが閉じ、変更箇所へ自動スクロール+差分フラッシュ
- フェーズナビは横スクロールチップ(現在フェーズをbrassドットで示す)
- 根拠表示はフルスクリーンモーダル(原文+ハイライト+「発見に戻る」)
- タップターゲット44px以上。レバー行・簿外行は行全体をタップ領域に
- シナリオ注記はサマリー直下に1行

### 2-3. 共通
- 1セクション=1カード=1メッセージ。カード見出しは「診断 — この会社の型」のように役割を名詞で
- 本文が3行を超えるAI出力は自動的に「…続きを読む」で折りたたみ
- **二層の見え方**: シナリオ層の結果とライブ層の結果を並べる場合、ライブ層側にだけ小さな「実解析」バッジ(情報色)を付け、能力の境界をUIで示す

## 3. フェーズ別 画面要件

各フェーズは4状態(初期/実行中/完了/エラー)を必ず定義する。エラーは既存の日本語正規化表を使用。

### P0 ソーシング(自社で入力)
- 入力は1画面1問のステップ形式ではなく1枚フォームだが、**入力に応じてprior チップがライブで並び替わる**(業種選択→即座に5チップ表示、依存度入力→該当チップがamberに点灯)。「入れるほど当たりがつく」を入力中から体感させる
- 完了状態: 「この型の要注意ポイント」カード+機密性注記「ここまで機密情報は一切使っていません」を明示(機密性プレゼンへの伏線)

### P1 資料投入・実解析(シナリオ層 + ライブ層)
- **シナリオ層**(サンプル企業): 決定的 findings を表示。注記「事前診断済みのシナリオです」を常設。ここで「いま全文を解析した」ように見せない
- **ライブ層**(給与CSV×勤怠のみ):
  - ドロップゾーン: 点線枠+「サンプル資料を投入」ボタン(同梱CSVをワンタップ投入。架空・結果既知・処理は本物)
  - **実行中=ライブ実況コンソール**(見せ方。演出可):
    - カード内に実況行がストリーム表示。1行=アイコン+動詞形の短文(「給与台帳 36ヶ月分を読込中…」「3月分に乖離を検出」)
    - 表示は最新5行、古い行は上へフェードアウト。完了時は「実況を見る(12件)」に畳む
    - 検出行のみsemantic色(注意=amber/重大=red)。それ以外は--muted
  - **完了時の能力証明**: 発見事項は実解析由来。「根拠を見る」で台帳の該当行へジャンプできること。完了と同時にタイムカウンターが+加算(400msカウントアップ+brassの微発光1回)
- 完了状態: 発見事項リスト(重要度ドット+推計レンジバー+「根拠を見る」)
- **EXIT予告タグ**(一気通貫の最安対処): 各発見事項カードに1行  
  `EXIT時: 処置済みとして開示可 → 評価ディスカウント回避`  
  スタイル: 12px/`--muted`、情報色の細い左線でも可。P5実装を待たない

### P2 Q&Aリスト・報告書生成
- 発見事項カードに「質問リストを生成」ボタン。生成中はスケルトン行(実況コンソール簡易版)
- 出力: テーブル(質問/カテゴリ/優先度チップ/依頼資料)。**ダウンロードボタンは`xlsx`拡張子まで表示**(「Excelでダウンロード(.xlsx)」)— 成果物が仕事の形であることをボタン文言で証明
- 報告書ドラフトは「Word(.docx)でダウンロード」+1ページ目のプレビューサムネイル表示

### P3 SPA論点(簿外債務と処置)
- 既存の主軸タブ連動を維持。処置カードに**処理方法の3択バッジ**(価格調整/エスクロー/是正投資)を表示し、選択で金額影響が差分フラッシュ
- PC: 3軸の処置を横並び比較できる「比較モード」トグル
- 問いカードの中身は `questions[phase3][strategy_axis]` から取得(主軸切替で問いも差し替わる)

### P4 レバー→実ツール接続(再生・導入寄りで最優先)
- レバー詳細展開内に「**このツールを試す →**」ボタン(secondaryスタイル、新規タブで該当デモへ)
- ボタン脇に稼働ステータス風の緑ドット+「デモ稼働中」— レバーが実在する証拠の演出
- **往路**: クエリに `caseId` / `strategy_axis` / `leverId` / `scrollY` / `leverExpanded=1` 等を付与
- **戻り導線(受け入れ必須)**:
  - 各ツールデモ側ヘッダーに「DD画面に戻る」
  - 復帰時、**該当レバーカードが展開したまま**、**スクロール位置を復元**する
  - 商談の流れが物理的に止まらないこと。展開忘れ・先頭復帰は不合格

### P5 EXIT準備
- 処置済み/未処置をチェックリスト表示(処置済みはbrassのチェック)。進捗リング「売却準備 68%」
- 「IM章立てをドラフト」ボタン→買い手類型タブ別に章立てが変わる
- 初回商談では未実装でも可(P1のEXIT予告タグ+口頭でカバー)

### 横断A: 返した時間カウンター(シグネチャー)
- PC: 右上固定。`⏱ 返した時間 14.5h` 形式。クリックで内訳ポップ
- SP: stickyサマリーバー右端にコンパクト表示(`14.5h`)。タップで同内訳
- **加算タイミング: 各機能の完了時のみ**(開始時加算禁止)
- **内訳ポップ必須項目**:
  - 機能別の代行時間一覧
  - **算定根拠**: `標準的なDD実務工数(○○を参照)に基づく実務時間換算の試算`  
    ※参照先ドキュメント名は実装時に固定し、聞かれたら即答できる一文にする
  - フッターに小さく「試算です」
- 加算アニメーションは数値ロールアップのみ。**音・紙吹雪・バウンスは禁止**(高級感を壊す)
- デモ終了時: 「このセッションでAIが代行した作業: 約14.5時間」のサマリーカードを最下部に表示(商談の締めと連動)
- 内訳を開いたときに根拠が出ること自体が「作り込んだ」証明になる

### 横断B: あなたへの問いカード
- 各フェーズ出力の最後に固定。デザインは他カードと差別化: `--ink`背景+白文字+brassの細い上線(委員会モード)
- 見出し固定文言「ここから先は、経営判断です」+問い2〜3個(番号なし・短文)
- **データ源**: 企業JSON `questions[phase][strategy_axis]` の在庫。AI都度生成禁止(レバーと同じ思想)
- 主軸タブ変更時は問いも差し替え。フェーズ固有の判断軸に縛り、全フェーズ同一文の使い回し禁止
- 唯一、AI出力の中で「答えではなく問い」を置く場所であることをスタイルで示す

### 横断C: 商談タイプ別の画面フォーカス
| タイプ | UIで前面に出すもの |
|---|---|
| 再生・導入寄り | P4レバー詳細・「このツールを試す」・戻り状態保持 |
| DD能力寄り | P1ライブ層ドロップゾーン・実解析バッジ・根拠ジャンプ |

## 4. 状態設計の共通ルール

| 状態 | ルール |
|---|---|
| 実行中 | 無言スピナー禁止。必ずライブ実況(P1仕様)か、最低でも「何をしているか」1行+進捗。実況は演出可だが、シナリオ層では能力偽装にならない注記を維持 |
| 空 | 招待文+主ボタン(「サンプル資料を投入して、AIの働きを見る」)。謝罪調・空白のみ禁止 |
| エラー | 日本語正規化表の文言+復帰ボタン。技術文字列の生表示禁止 |
| 回数制限(Managed Trial) | 残回数を事前に小さく常設表示(「体験残り 7回」)。突然のブロックにしない |
| 差分 | §1-3フラッシュ仕様。**比較対象は「直前の生成結果」のみ**(2つ前とは比較しない) |
| シナリオ明示 | サンプル企業モード中は「事前診断済みのシナリオです」が常に見えること |

## 5. 品質基準・アクセシビリティ

- タップ44px以上/フォーカスリング可視/`prefers-reduced-motion`対応(フラッシュ・ロールアップ停止)
- コントラスト: 本文4.5:1以上。brassは大サイズ(18px+)専用。注記(`--muted`)も背景対比で可読
- 数値の等幅(tnum)・単位の統一(億/百万円の混在禁止: 1億未満のみ百万円)
- レスポンシブ検証ポイント: 375px(SP基準)/768px/1024px/1440px
- 体感速度: パラメータ→フロント計算の反映は100ms以内。AI再生成は「提案を更新」ボタンでのみ実行(誤爆課金防止と、更新の儀式感の両立)

## 6. 受け入れチェックリスト(実装完了の定義)

### 6-1. 先行(商談投入の定義) — 全面リデザイン不要
- [ ] 「返した時間」カウンターが表示され、**完了時のみ**加算される
- [ ] 内訳ポップに機能別時間と**算定根拠(試算である旨)**が出る
- [ ] 「あなたへの問い」カードがフェーズ出力末尾にあり、`questions`在庫(フェーズ×主軸)から出る
- [ ] サンプル企業選択時に「事前診断済みのシナリオです」注記が見える
- [ ] (再生寄り) 各レバーから既存デモが開き、**戻り後にレバー展開状態とスクロール位置が復元される**
- [ ] (DD寄り) ライブ層: 同梱給与CSV×勤怠の**実解析**が走り、根拠行へジャンプできる
- [ ] 発見事項にEXIT予告タグ1行がある

### 6-2. 全面UI(受注後)
- [ ] PCで、左レールのスライダー操作と右の数値変化が同一視界で確認できる
- [ ] SPで、条件変更→反映→該当箇所へ自動スクロール+フラッシュまでが1フローで動く
- [ ] すべてのAI実行中画面に実況または進捗説明があり、無言スピナーが1つもない
- [ ] Q&AリストがExcel、報告書がWordで実際にダウンロードできる
- [ ] 金額の桁ずれ・等幅崩れがない(tnum適用確認)
- [ ] reduced-motion環境で差分がバッジのみで伝わる
- [ ] 375px幅で横スクロールが発生しない
- [ ] シナリオ層とライブ層の境界がUI上で区別できる(注記または「実解析」バッジ)
