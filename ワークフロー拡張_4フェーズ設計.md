# ワークフロー拡張 ―― 4フェーズ・バリューアップ・モデル

> AXEON デモを「簿外債務発見」から **DD支援 → PMI → バリューアップ → EXIT** の一気通貫ワークフローに拡張するための設計書。
> 同一の対象企業データ（5業種ペルソナ）が全フェーズを貫き、DD で確定した結果が後続フェーズへ自動連携する。

---

## 1. 4フェーズの目的と見せ方

| フェーズ | タイミング | デモで見せること | 成果 |
|---------|-----------|----------------|------|
| **① DD支援** | 買収前 | 定量査定（機械的推計）＋発見系（突合・言語・時系列）／人間検証 | 買収判断の精度向上 |
| **② PMI** | 最初の100日 | DD確定フラグ → 是正タスク（担当・期日・解消額） | のれん毀損防止 |
| **③ バリューアップ** | 保有期間 | 実態EBITDA補正 → 労務適正化＋業種別AI実装 | 企業価値そのものを上げる |
| **④ EXIT** | 売却前 | EVブリッジ（Entry → 改善 → 再評価 → Exit） | 高いマルチプルで売れる |

### データ連携フロー

```
対象企業DB（5業種ペルソナ）
    ↓
[DD] 定量査定 + 発見系 → 人間検証（採用/要確認/誤検知）
    ↓ 確定簿外債務・補正額
[PMI] 是正タスク化（100日プラン）
    ↓ 解消額・実態補正
[バリューアップ] 実態EBITDA → 施策 → 改善後EBITDA
    ↓
[EXIT] Entry EV → EBITDA改善 → マルチプル再評価 → Exit EV
```

## 2. 定量査定 vs 発見系の区別

| 区分 | 定義 | 代表例 | UI表示 |
|------|------|--------|--------|
| **定量** (`cat: "定量"`) | 勤怠・給与・社保・就業規則・従業員名簿から**ロジックで金額を推計**できる | 未払残業代、社保未加入、退職給付引当不足、未消化有給引当 | 「計算の中身」(`calc`) を表示 |
| **発見** (`cat: "発見"`) | 複数ソースの矛盾・言語シグナル・時系列異常から**気づく**類型 | CoC条項、簿外保証、偶発債務、業種特有リスク | 根拠チェーン（既存UI） |

**鉄則**: 定量は再現可能な計算式を必ず付与。発見は説明可能性（根拠ソース）を重視。

```js
  },

```

### 3.2 新規 `pmi` オブジェクト

```js

  headline: "最初の100日 ― 属人化の解体・データ基盤・標準化",
```

### 3.4 新規 `exit` オブジェクト

```js
exit: {
  entryEbitda: 5500,           // 万円。DD補正後実態EBITDA
  entryMult: 5.5,
  exitEbitda: 9500,            // 万円。バリューアップ後
  exitMult: 10.5,
  story: "テック-enabled 物流プラットフォームとしての売却ストーリー",
  bridge: [
    { label: "Entry EV（実態EBITDA × マルチプル）", amt: 30250 },

  ],

```

> 金額はすべて **万円** 単位。EV = EBITDA × マルチプル（デモ簡略化）。

## 4. ランタイム状態と派生ロジック

### 4.1 `state.js` 拡張

- `currentPhase`: `"dd" | "pmi" | "valueup" | "exit"`
- 既存: `ran`, `judgments`, `extraPriors`, `scanning`（ペルソナ別）

### 4.2 `derive.js` 派生関数

| 関数 | 入力 | 出力 |
|------|------|------|
| `getConfirmedFlags(persona, state)` | judgments === "ok" | 確定フラグ一覧＋金額 |
| `getConfirmedLiability(persona, state)` | 同上 | `{ lo, hi }` 万円 |
| `getPmiTasks(persona, state)` | 確定フラグ id | フィルタ済み PMI タスク |
| `getValueupWaterfall(persona, state)` | DD確定額 | 報告→補正→施策→改善後EBITDA |
| `getExitBridge(persona, state)` | valueup結果 | EVブリッジ各段 |

**連動ルール**: DD で「採用」したフラグのみ PMI タスクに表示。確定額が変わるとバリューアップ・EXIT の数値も再計算。
